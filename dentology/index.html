Have you worked on any personal or side projects outside of your main roles?

If so, tell us about one and include any GitHub links, demos, or case studies.

Short Answer: Yes, proud of are Yelix and AAD. Yeah, I've worked on a bunch of side projects. Two that I'm particularly proud of are Yelix and AAD. Yelix is probably the one I'm most excited about. It's an abstraction layer I built on top of Hono using TypeScript and Deno. The idea was to enhance the API development experience while keeping full compatibility with Hono's core features. So it doesn't replace Hono - it extends it with things like automatic OpenAPI documentation generation, an enhanced middleware system with execution time tracking, seamless Zod integration for request validation, built-in Scalar UI support for beautiful API docs, better logging with request/response tracking, and improved TypeScript type safety. It's been a really fun project to work on, and it actually won some recognition - got a silver medal in the Karadeniz region competition and then a gold medal nationwide in Turkey's 1Fikir1Türkiye competition. You can check it out on GitHub at github.com/yelix-org/yelix. The other project I want to mention is AAD, which is a Chrome extension I built for customizing GitHub's interface. I got frustrated with how GitHub's homepage worked, especially on my ultra-wide monitor, so I decided to build something that would make it way more customizable and productive. It lets you set up 16 different containers, supports GitHub's dark-dimmed theme, and gives you way more API requests if you use a personal access token - like 5000 per hour instead of the default 60. It has this focus mode where you can open issues and repos in modals and comment right there, plus drag and drop to reorder everything however you want. The coolest part is that it's actually being used by multiple companies in production, which feels pretty awesome. It's available on the Chrome Web Store, and I made sure to keep it privacy-first - metrics are optional and it only logs exceptions if you give permission. Both of these projects taught me a lot about building things that people actually use, not just building for the sake of it. Yelix helped me understand abstraction layer design, API documentation, and how to extend existing frameworks while maintaining compatibility, while AAD taught me about building stable extensions that work across GitHub's UI changes and respecting user privacy from the ground up. I'm also pretty active in open-source. I'm a member of the Express.js Performance Working Group, and I'm working towards joining the Express.js triage team as soon as possible. I've been contributing quite a bit to Express.js, focusing on performance improvements and tooling. It's been really rewarding to work on something that so many developers rely on.
